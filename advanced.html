Materi Pemrograman Lanjutan

1. Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) adalah paradigma pemrograman yang berfokus pada objek dan data, bukan logika. OOP mencakup empat prinsip utama: enkapsulasi, pewarisan, polimorfisme, dan abstraksi.

Konsep Dasar OOP

  • Enkapsulasi: Menyembunyikan implementasi internal dan hanya menunjukkan antarmuka yang dibutuhkan.
  • Pewarisan: Mengizinkan kelas untuk mewarisi properti dan metode dari kelas lain.
  • Polimorfisme: Memberikan kemampuan untuk mendefinisikan beberapa bentuk dari metode yang sama.
  • Abstraksi: Menyembunyikan detail implementasi dan hanya menampilkan fungsionalitas yang diperlukan.

Contoh OOP di JavaScript

class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(this.name + ' makes a sound');
  }
}

class Dog extends Animal {
  constructor(name) {
    super(name); // Panggil constructor dari kelas induk
  }

  speak() {
    console.log(this.name + ' barks');
  }
}

let dog = new Dog('Buddy');
dog.speak(); // Output: Buddy barks

Contoh OOP di PHP


class Animal {
  protected $name;

  public function __construct($name) {
    $this->name = $name;
  }

  public function speak() {
    echo $this->name . " makes a sound\n";
  }
}

class Dog extends Animal {
  public function speak() {
    echo $this->name . " barks\n";
  }
}

$dog = new Dog("Buddy");
$dog->speak(); // Output: Buddy barks

2. Pemrograman Asinkron Lanjutan

Pemrograman asinkron memungkinkan Anda untuk menangani operasi IO secara non-blocking, yang sangat penting dalam pengembangan aplikasi web modern. Di sini kita akan membahas lebih lanjut tentang Promises dan Async/Await.

Promises di JavaScript

function fetchData() {
  return new Promise((resolve, reject) => {
    let success = true; // Coba ganti ke false untuk melihat penanganan error
    setTimeout(() => {
      if (success) {
        resolve("Data berhasil diambil!");
      } else {
        reject("Terjadi kesalahan dalam mengambil data.");
      }
    }, 2000);
  });
}

fetchData()
  .then((data) => console.log(data)) // Output: Data berhasil diambil!
  .catch((error) => console.log(error)); // Output jika gagal

Async/Await di JavaScript

async function getData() {
  try {
    let response = await fetchData();
    console.log(response); // Output: Data berhasil diambil!
  } catch (error) {
    console.log(error); // Output jika terjadi error
  }
}

getData();

Asynchronous di PHP menggunakan cURL


function fetchData($url) {
  $ch = curl_init($url);
  curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
  $response = curl_exec($ch);
  curl_close($ch);
  return $response;
}

$data = fetchData('https://jsonplaceholder.typicode.com/posts');
echo $data;

3. Desain Pattern dan Prinsip SOLID

Desain Pattern adalah solusi umum untuk masalah yang sering terjadi dalam pengembangan perangkat lunak. Sementara itu, prinsip SOLID adalah lima prinsip penting dalam OOP yang membantu meningkatkan desain dan pemeliharaan kode.

Contoh Desain Pattern: Singleton

class Singleton {
  static $instance = null;

  private function __construct() {}

  public static function getInstance() {
    if (self::$instance === null) {
      self::$instance = new Singleton();
    }
    return self::$instance;
  }
}

$instance1 = Singleton::getInstance();
$instance2 = Singleton::getInstance();

var_dump($instance1 === $instance2); // Output: bool(true), karena hanya ada satu instance

Prinsip SOLID

  • S - Single Responsibility Principle (SRP): Setiap kelas harus memiliki tanggung jawab tunggal.
  • O - Open/Closed Principle (OCP): Kode harus terbuka untuk ekstensi, tetapi tertutup untuk modifikasi.
  • L - Liskov Substitution Principle (LSP): Kelas turunan harus dapat menggantikan kelas induk tanpa merusak fungsionalitas program.
  • I - Interface Segregation Principle (ISP): Jangan memaksa klien untuk bergantung pada metode yang tidak mereka gunakan.
  • D - Dependency Inversion Principle (DIP): Bergantung pada abstraksi, bukan pada implementasi.
